# 安全政策 | Security Policy

## 🔒 安全报告 | Reporting a Vulnerability

我们非常重视衍智体 (YANZHITI) 项目的安全性。如果您发现安全漏洞，**请不要公开 Issue**，请通过以下方式私密报告：

### 报告方式 | How to Report

1. **发送邮件至**: `yanzhiti-security@proton.me`
2. **邮件标题格式**: `[SECURITY] 漏洞简述 - 日期`
3. **请包含以下信息**:
   - 漏洞类型和描述
   - 复现步骤
   - 影响范围
   - 建议修复方案（如有）

### 响应承诺 | Response Commitment

| 阶段 | 时间范围 |
|------|---------|
| 确认收到 | 48 小时内 |
| 初步评估 | 7 天内 |
| 修复方案 | 严重漏洞 14 天内 |
| 公开披露 | 修复发布后 |

## 🛡️ 已知安全措施 | Security Measures

### API 密钥保护
- ✅ API Key 仅存储在本地内存中，不写入日志
- ✅ 不将 API Key 提交到版本控制系统
- ✅ 支持 `.env` 文件或环境变量配置
- ⚠️ 当前 Web UI 使用 localStorage 存储（生产环境需加密）

### 输入验证
- ✅ 所有 API 输入使用 Pydantic 进行类型校验
- ✅ 温度参数限制在 [0, 2] 范围
- ✅ Token 数限制在合理范围内
- ✅ SQL 注入防护（本项目无数据库操作）

### 网络安全
- ✅ Web UI 默认绑定 `0.0.0.0:8000`（可配置）
- ✅ CORS 中间件已启用（生产环境应限制域名）
- ✅ WebSocket 连接支持会话管理

### 代码安全
- ✅ Ruff linter 自动检测安全问题
- ✅ 无硬编码密钥或凭证
- ✅ 无 eval() 或 exec() 调用
- ✅ 依赖项定期更新检查

## 🚫 不被视为漏洞的情况 | Not Considered Vulnerabilities

以下情况通常不被视为安全漏洞：
- 需要用户主动输入 API Key 才能触发的功能问题
- 信息泄露但需要已认证访问的问题
- 已在文档中说明的已知限制
- 第三方依赖库的问题（我们会跟进上游修复）
