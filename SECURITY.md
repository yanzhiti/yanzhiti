# 🔒 安全策略 | Security Policy

## 报告安全问题

如果您发现衍智体 (YANZHITI) 的安全漏洞，请负责任地披露。

### 报告方式

**请通过以下方式报告安全问题**:

- 📧 邮箱：contact@yanzhiti.com
- 🔐 使用 GitHub Private Vulnerability Reporting (推荐)

**请勿**:
- ❌ 在公开 Issue 中报告安全问题
- ❌ 在社交媒体上披露
- ❌ 在未修复前公开讨论

### 报告内容

为了帮助我们更好地理解问题，请提供:

1. **问题描述** - 清晰描述安全漏洞
2. **复现步骤** - 详细的复现方法
3. **影响范围** - 可能受影响的版本和功能
4. **环境信息** - OS、Python 版本等
5. **修复建议** (可选) - 如有想法欢迎分享

### 响应时间

我们承诺:

- ⏱️ **24 小时内** - 确认收到报告
- ⏱️ **48 小时内** - 初步评估和响应
- ⏱️ **7 天内** - 修复计划时间表
- ⏱️ **30 天内** - 目标修复时间 (严重漏洞)

### 处理流程

```
报告 → 确认 → 评估 → 修复 → 测试 → 发布 → 披露
```

1. **报告** - 收到安全报告
2. **确认** - 确认问题存在
3. **评估** - 评估影响范围和严重程度
4. **修复** - 开发修复方案
5. **测试** - 全面测试修复
6. **发布** - 发布安全更新
7. **披露** - 公开披露 (经报告者同意)

### 披露政策

我们遵循负责任的披露原则:

- 在修复发布前不公开披露
- 给予报告者适当的感谢 (经同意)
- 在安全公告中说明问题 (不泄露敏感信息)

### 安全最佳实践

#### 对于用户

1. **API 密钥保护**
   - 不要将 API 密钥提交到版本控制
   - 使用环境变量或 .env 文件
   - 定期轮换密钥

2. **权限控制**
   - 使用最小权限原则
   - 审查工具执行权限
   - 监控异常行为

3. **数据安全**
   - 敏感数据本地存储
   - 加密重要配置
   - 定期备份

#### 对于贡献者

1. **代码安全**
   - 验证所有用户输入
   - 避免硬编码凭据
   - 使用安全的 API

2. **依赖管理**
   - 定期更新依赖
   - 审查依赖安全性
   - 避免不必要的依赖

3. **测试覆盖**
   - 包含安全测试
   - 测试边界情况
   - 模拟攻击场景
