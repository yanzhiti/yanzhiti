## 💡 最佳实践

1. **有意义的提交信息** - 说明为什么修改，不只是修改了什么
2. **小步提交** - 频繁提交，每次提交一个逻辑变更
3. **提交前检查** - 运行测试和 lint
4. **使用分支** - 新功能在分支开发，完成后合并
